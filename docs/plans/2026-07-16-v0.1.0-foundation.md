### Phase 3 — Naming convention

**T-10** `app/naming/convention.py` — template renderer
- Accept a `Track` (or dict of tokens) and a template string.
- Universal default template: `{album_artist}/{year} - {album}/{disc_track} - {title}.{ext}`. Replace tokens: `{album_artist}`, `{album}`, `{year}`, `{disc_track}`, `{title}`, `{ext}`. `disc_track` is `TT` for one-disc releases and `D-TT` for multi-disc releases; track numbers are two digits.
- Sanitise each token: strip leading/trailing whitespace, replace `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|` with `_`, collapse runs of `_`, limit segment length to 200 chars.
- Test: nominal case, missing token falls back to `Unknown`, path traversal attempt (`../`) is neutralised, long token truncated.

**T-11** `app/naming/convention.py` — path traversal guard
- After rendering, resolve the path relative to `LIBRARY_ROOT` and assert it does not escape.
- Raise `NamingError` if it would escape.
- Test: crafted token containing `..` is blocked.

## 3. Provider Contracts

### Health contract

```
GET /health → 200
{
  "status": "ok" | "degraded" | "down",
  "sources": {
    "slskd":    {"available": true|false, "reason": null|"..."},
    "prowlarr": {"available": true|false, "reason": null|"..."},
    "sabnzbd":  {"available": true|false, "reason": null|"..."},
    "youtube":  {"available": true|false, "reason": null|"..."},
    "tidal":    {"available": false, "reason": "unavailable in v0.1.0"}
  }
}
```

TIDAL is always reported as `available: false` with a fixed reason string. No TIDAL code runs.

### Search contract

```
POST /search
{ "query": "...", "sources": ["slskd", "youtube"] }

→ 200
{
  "results": [
    {
      "source": "slskd",
      "title": "...", "artist": "...", "album": "...",
      "duration_sec": 214,
      "size_bytes": 8294400,
      "format": "flac",
      "url": "...",
      "metadata": {}
    }
  ],
  "source_states": {
    "slskd": {"available": true, "reason": null}
  }
}
```

## 4. Naming Security Checklist

Before computing a path preview, every naming token must pass:

- [ ] Strip leading/trailing whitespace
- [ ] Reject or replace filesystem-unsafe characters: `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`, null byte
- [ ] Replace `.` sequences (`..`, `...`) with `_`
- [ ] Collapse repeated `_` or whitespace to single
- [ ] Truncate to 200 characters per path segment
- [ ] After template rendering, `Path(LIBRARY_ROOT, rendered).resolve()` must start with `Path(LIBRARY_ROOT).resolve()`
- [ ] If resolution escapes root: raise `NamingError`, do not store preview

## 5. Source Capability State Rules

- Each `SourceAdapter` determines its own availability at health-check time via a real network call.
- A source that returns `available=False` is excluded from search fan-out silently; the response includes its state.
- TIDAL hardcodes `available=False, reason="unavailable in v0.1.0"`. No code path enables it.
- Capability states are never cached beyond the lifetime of a single request in v0.1.0.

---

## 6. AcoustID / fpcalc Integration Details

```
fpcalc -json <audio_file>
→ {"duration": 214.3, "fingerprint": "AQAA..."}
```

- Invoked via `asyncio.create_subprocess_exec` to avoid blocking the event loop.
- Timeout: 30 seconds per file.
- If `fpcalc` is not found in PATH: log `WARNING fingerprinting skipped: fpcalc not found`, set `fingerprint_state=skipped`.
- If file is unreadable or fpcalc exits non-zero: set `fingerprint_state=failed`.
- AcoustID lookup: only performed if `ACOUSTID_API_KEY` is set and fingerprinting succeeded. Returns list of candidate MBIDs sorted by score; top candidate is used if score ≥ 0.6.

---

## 7. MusicBrainz Rate Limiting

MusicBrainz allows 1 request/second for non-authenticated clients. Implementation:

- Global `asyncio.Semaphore(1)` + `asyncio.sleep(1.1)` after each release.
- `User-Agent` header must be `{APP_NAME}/{APP_VERSION} ({CONTACT})` — required by MusicBrainz policy.
- Retry on HTTP 503 with exponential backoff: 2s, 4s, 8s. After 3 failures, raise `MetadataError`.
